# 🚗 VDAS-Insight

**Vehicle Data Analysis & Statistics** — 大規模車両計測データ（J1939 等）の前処理済み
parquet / csv を対象とした、研究・業務向けの可視化・統計・**コホート比較**プラットフォーム。

1 データの統計・時系列分析はもちろん、複数のデータを **タグ（集団）** としてまとめ、
タグ同士を **レート正規化**（時間あたり・距離あたり・割合）で比較できます。N（データ数・
計測時間・走行距離）が異なるコホートでも公平に比較できるのが最大の特徴です。

## 主な機能

| 機能 | 内容 |
|------|------|
| **スキーマ非依存** | 列名に依存せず、各列に *役割*（時間 / ギア / フラグ / 車速 / 数値 / カテゴリ）を割り当て。役割は自動推定され、手動修正も可能。 |
| **単一データ分析** | 記述統計・ヒストグラム・散布図・多信号時系列（信号ごとのサブプロット）。 |
| **ギア段分析** | 変速検出（アップ / ダウン）、変速回数・時間あたり・距離あたり、ギア段別滞在時間、変速の遷移行列（from→to ヒートマップ）。 |
| **フラグ（0/1）分析** | 立上り回数、ON 率（デューティ）、ON 時間分布、立上り間隔分布。 |
| **2D マップ（運転領域マップ）** | 任意の X×Y 信号（例: エンジン回転×車速）を格子集計し、色で **滞在割合 / サンプル数 / 平均 / 標準偏差** を表示（EX-Summary 風のコンター）。符号付き量（平均加速度など）は 0 中心の発散配色（減速=青／加速=赤）。 |
| **コホートマップ / 差分マップ** | タグ（コホート）全体を共通グリッドにプールした運転領域マップ、および **2 コホートの差分マップ (A−B)**（例: city は低速側で赤=多い、highway は高速側で青=多い）。集計条件（車速≤70 等）も適用可。 |
| **サマリシート** | 選んだ指標を**ミニ棒グラフのマトリクス**で一覧（EX-Summary 風スモールマルチプル）。各パネルがコホート比較になり、多数の指標×コホートを一目で俯瞰。指標・コホート・列数を選択可能。 |
| **派生信号** | 既存信号から新しい信号を算出して一覧に追加。車速→**加速度(m/s²)・加加速度(m/s³)**、汎用の微分/2階微分/移動平均/絶対値。派生信号を元にさらに派生も可能（連鎖）。定義は DuckDB に保存され、時系列・統計の対象になります。 |
| **しきい値イベント** | 信号に条件（`x>閾値` / `x<閾値` / `|x|>閾値`、平滑化窓でデバウンス）を設定して **0/1 のイベント信号**を生成（例: 急減速 `|accel|>2.5`）。フラグ分析タブとコホートのフラグ指標（回数・時間あたり回数・ON率）に自動反映。 |
| **タグ / コホート** | データセットを集団としてタグ付け（多対多）。**カテゴリ**（メーカー・走行条件など）でタグを整理でき、1 データに複数タグを付与可能。 |
| **一括タグ・データ管理** | 読み込み時にまとめてタグ付与。**データ管理**画面で複数選択して **タグの一括付与/解除・一括削除**。 |
| **コホート比較** | プール値（全体合計から算出したレート）＋ データ単位のばらつき（点オーバーレイ）＋ 基準コホート=100 の相対指数。**タグの AND 絞り込み**と **カテゴリで分割**に対応（例:「高速条件に絞ってメーカーごとに比較」）。指標はギア/フラグに加え、**数値・派生信号の平均/標準偏差/最大/最小**（平均は N で長さ重み付けした正しいプール値）や**イベント回数**も選択可。CSV エクスポート対応。 |
| **条件付き集計** | サンプル単位の条件（例: `車速 ≤ 70`、複数 AND 可）を設定すると、条件成立中のサンプル/イベントだけで集計。**変速回数は条件成立中に発生した分のみ計上**、時間あたりは条件成立時間で正規化、距離・フラグ・数値統計も条件成立部分のみ。**コホート比較・サマリ**の両タブで利用でき、条件は**名前付きプリセット**として保存・再利用可能。 |
| **計測ツール的UI** | INCA / CANape 風のライトテーマ・ドッキング可能パネル。pyqtgraph による高速・リアルタイム描画、リンク軸・クロスヘアカーソル（大規模データも自動ダウンサンプルで軽快）。 |
| **永続化** | データセット登録・役割・タグを DuckDB に保存。 |

## アーキテクチャ

```
                +-------------------+
   parquet/csv →|  DuckDB (engine)  |→ 直接クエリ（コピー不要・高速）
                +-------------------+
                          │
   +----------------------┴-----------------------+
   │  vdas/ （解析エンジン・純Python・UI非依存）   │
   │   datasets  役割マッピング / スキーマ         │
   │   tags      コホート（集団）                  │
   │   analysis  core / metrics / gears / flags /  │
   │             groups（プール集計・相対比較）    │
   +----------------------┬-----------------------+
                          │
          +---------------┴-----------------+
          │  desktop/  PySide6 + pyqtgraph  │  ドッキング可能なパネル群 +
          │  （INCA/CANape 風の計測クライアント）│  計測/ギア/フラグ/統計/コホート比較タブ
          +---------------┬-----------------+
                          │
                +---------┴---------+
                │  DuckDB (metadata)│  datasets / column_roles / tags / dataset_tags
                +-------------------+
```

- **DuckDB は二役**：parquet / csv を直接読む解析エンジンであり、メタデータ（データセット
  登録・役割・タグ）の永続ストアでもあります。バルクデータをコピーしません。
- **解析エンジン（`vdas/`）は UI 非依存**の純 Python。CLI / ノートブックからも利用可能で、
  デスクトップ UI（`desktop/`）はこのエンジンの薄いフロントエンドです。

### 公平な比較の考え方（重要）

指標を 2 種類に分けて扱います:

- **示量的（extensive）** … 加算できる量（変速回数・計測時間・走行距離・ON 時間など）。
  コホートでは **合計** してプールします。
- **示強的（intensive）** … レート・割合・平均。コホートでは **プール後の示量量から再計算**
  します（各データのレートを単純平均しない）。これにより 2 分のログと 5 時間のログを
  同等に扱うことを避け、`総変速回数 ÷ 総時間` のような正しい代表値になります。

さらに **相対指数**（基準コホート=100）で「A に対して B は時間あたり 118%」のように、
N が違っても割合で読み取れます。

## クイックスタート

```bash
pip install -r requirements.txt

# （任意）J1939 風のサンプルデータを生成
python scripts/gen_sample.py

# デスクトップアプリ起動
./run.sh
#   または: python -m desktop.main
```

> **Linux で Qt に必要なシステムライブラリ**（ヘッドレス/最小環境の場合）:
> `sudo apt-get install -y libegl1 libgl1 libxkbcommon0 libdbus-1-3 libglib2.0-0 libfontconfig1`
> Windows / macOS は pip の PySide6 だけで動作します。

起動後の使い方:

1. ツールバー **＋ 登録 → 同梱サンプルを一括登録**（city / highway タグが自動付与）。
2. 左の **データセット** で対象を選び、**信号** パネルで表示信号にチェック → **計測(時系列)** タブに描画。
3. **ギア段 / フラグ / 統計** タブで単一データを分析。
4. **コホート比較** タブで city ↔ highway を変速回数/時間、変速/km、フラグ ON 率などで比較。

**一括タグ付け / データ管理**: ツールバーの **🗂 データ管理** を開き、複数行を Shift / Ctrl で
選択 → タグの一括付与・解除、または一括削除。読み込み直後にも「まとめて付与」ダイアログが出ます。
タグはカテゴリ（メーカー・走行条件など）で整理でき、1 データに複数付けられます。

**コホート比較（組み合わせ）**: コホート比較タブで
- **作り方=カテゴリで分割** → 例「メーカー」で分割し、メーカーごとのコホートを自動生成
- **絞り込み(AND)** → 例「高速」を選ぶと、各コホートを高速データだけに限定
これで「高速条件に絞って、メーカー同士を変速回数/時間などで比較」がそのまま行えます。

**派生信号の作り方**: 信号パネルの **ƒ 派生信号…** から元信号と種類を選ぶだけ。車速を選ぶと
既定で「加速度(m/s²)」、続けて「加加速度(m/s³)」も追加できます（微分はノイズを増幅するため
平滑化窓を指定可能）。追加後は一覧の **派生 (DERIVED)** グループに現れ、チェックで時系列に表示、
統計タブにも反映されます。右クリックで削除。

### 独自データを使う

- `sample_data/` に依存しません。**＋ 登録**（ファイル選択 / パス・グロブ）で parquet / csv を登録します。
- 登録時に列の役割を自動推定します。うまく当たらない列は右の **役割 / プロパティ** で修正
  できます（時間列は 1 つ、単位は s / ms / µs / ns / min から選択）。
- 時間列が無い場合は行インデックスを 1 Hz とみなしてレートを計算します（後から実時間列を
  設定可能）。

## データの期待形式

前処理済みの **1 行 = 1 サンプル（時系列）** の parquet / csv を想定します。列名は任意で、
役割で解釈します。サンプル（`scripts/gen_sample.py`）の列例:

| 列 | 役割 | 例 |
|----|------|----|
| `timestamp_s` | 時間 | 経過秒 |
| `current_gear` | ギア | 0=N, 1..6 |
| `vehicle_speed_kph` | 車速 | km/h（距離・km あたり指標に使用） |
| `engine_speed_rpm` | 数値 | rpm |
| `brake_switch`, `cruise_active`, `dpf_regen_active` | フラグ | 0/1 |
| `coolant_temp_c` | 数値 | ℃ |

> 自動推定は、車速（`speed`/`kph`…）とエンジン回転（`rpm`/`engine`…）を区別します。

## ライブラリ選定

- **PySide6 (Qt)** … 計測・解析系デスクトップの標準。ドッキング可能パネル、ネイティブな
  ツリー/テーブル、ライトテーマで INCA / CANape 的な操作感。
- **pyqtgraph** … 高速・リアルタイム描画。リンク軸のオシロスコープ表示、クロスヘア、
  大規模データの自動ダウンサンプルなど、計測ツールに最適。
- **DuckDB** … parquet/csv を高速に直接クエリでき、タグ等の永続化も 1 ファイルで完結。
