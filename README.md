# 🚗 VDAS-Insight

**Vehicle Data Analysis & Statistics** — 大規模車両計測データ（J1939 等）の前処理済み
parquet / csv を対象とした、研究・業務向けの可視化・統計・**コホート比較**プラットフォーム。

1 データの統計・時系列分析はもちろん、複数のデータを **タグ（集団）** としてまとめ、
タグ同士を **レート正規化**（時間あたり・距離あたり・割合）で比較できます。N（データ数・
計測時間・走行距離）が異なるコホートでも公平に比較できるのが最大の特徴です。

---

## 主な機能

| 機能 | 内容 |
|------|------|
| **スキーマ非依存** | 列名に依存せず、各列に *役割*（時間 / ギア / フラグ / 車速 / 数値 / カテゴリ）を割り当て。役割は自動推定され、手動修正も可能。 |
| **単一データ分析** | 記述統計・ヒストグラム・散布図・多信号時系列（信号ごとのサブプロット）。 |
| **ギア段分析** | 変速検出（アップ / ダウン）、変速回数・時間あたり・距離あたり、ギア段別滞在時間、変速の遷移行列（from→to ヒートマップ）。 |
| **フラグ（0/1）分析** | 立上り回数、ON 率（デューティ）、ON 時間分布、立上り間隔分布。 |
| **タグ / コホート** | データセットを集団としてタグ付け（多対多）。 |
| **コホート比較** | プール値（全体合計から算出したレート）＋ データ単位のばらつき（箱ひげ）＋ 基準コホート=100 の相対指数。CSV エクスポート対応。 |
| **インタラクティブ** | すべて Plotly（ズーム・パン・ホバー・PNG 書き出し）。 |
| **永続化** | データセット登録・役割・タグを DuckDB に保存。 |

## アーキテクチャ

```
                +-------------------+
   parquet/csv →|  DuckDB (engine)  |→ 直接クエリ（コピー不要・高速）
                +-------------------+
                          │
   +----------------------┴-----------------------+
   │  vdas/ （解析エンジン・純Python）             │
   │   datasets  役割マッピング / スキーマ         │
   │   tags      コホート（集団）                  │
   │   analysis  core / metrics / gears / flags /  │
   │             groups（プール集計・相対比較）    │
   │   viz       Plotly チャート                   │
   +----------------------┬-----------------------+
                          │
                +---------┴---------+
                │  app/ Streamlit   │  Home / Datasets / Explore / Tags / Compare
                +-------------------+
                          │
                +---------┴---------+
                │  DuckDB (metadata)│  datasets / column_roles / tags / dataset_tags
                +-------------------+
```

- **DuckDB は二役**：parquet / csv を直接読む解析エンジンであり、メタデータ（データセット
  登録・役割・タグ）の永続ストアでもあります。バルクデータをコピーしません。
- **解析エンジン（`vdas/`）は UI 非依存**の純 Python。CLI / ノートブックからも利用可能。

### 公平な比較の考え方（重要）

指標を 2 種類に分けて扱います:

- **示量的（extensive）** … 加算できる量（変速回数・計測時間・走行距離・ON 時間など）。
  コホートでは **合計** してプールします。
- **示強的（intensive）** … レート・割合・平均。コホートでは **プール後の示量量から再計算**
  します（各データのレートを単純平均しない）。これにより 2 分のログと 5 時間のログを
  同等に扱うことを避け、`総変速回数 ÷ 総時間` のような正しい代表値になります。

さらに **相対指数**（基準コホート=100）で「A に対して B は時間あたり 118%」のように、
N が違っても割合で読み取れます。

## クイックスタート

```bash
pip install -r requirements.txt

# （任意）J1939 風のサンプルデータを生成
python scripts/gen_sample.py

# アプリ起動
./run.sh
#   または: streamlit run app/Home.py
```

ブラウザで開いたら:

1. **Home** の「▶ サンプルを一括登録してタグ付け」でサンプルを取り込み（city / highway タグ付き）。
2. **Explore** で 1 データの統計・時系列・ギア・フラグを確認。
3. **Tags** でコホートを編集。
4. **Compare** で city ↔ highway を変速回数/時間、変速/km、フラグ ON 率などで比較。

### 独自データを使う

- `sample_data/` に依存しません。**Datasets** ページでファイルパス（グロブ可）または
  アップロードで parquet / csv を登録してください。
- 登録時に列の役割を自動推定します。うまく当たらない列は **Datasets → 役割の編集** で修正
  できます（時間列は 1 つ、単位は s / ms / µs / ns / min から選択）。
- 時間列が無い場合は行インデックスを 1 Hz とみなしてレートを計算します（後から実時間列を
  設定可能）。

## データの期待形式

前処理済みの **1 行 = 1 サンプル（時系列）** の parquet / csv を想定します。列名は任意で、
役割で解釈します。サンプル（`scripts/gen_sample.py`）の列例:

| 列 | 役割 | 例 |
|----|------|----|
| `timestamp_s` | 時間 | 経過秒 |
| `current_gear` | ギア | 0=N, 1..6 |
| `vehicle_speed_kph` | 車速 | km/h（距離・km あたり指標に使用） |
| `engine_speed_rpm` | 数値 | rpm |
| `brake_switch`, `cruise_active`, `dpf_regen_active` | フラグ | 0/1 |
| `coolant_temp_c` | 数値 | ℃ |

> 自動推定は、車速（`speed`/`kph`…）とエンジン回転（`rpm`/`engine`…）を区別します。

## 設定（環境変数）

| 変数 | 既定 | 説明 |
|------|------|------|
| `VDAS_DATA_DIR` | `./data` | メタDB・アップロードの保存先 |
| `VDAS_DB_PATH` | `$VDAS_DATA_DIR/vdas.duckdb` | DuckDB メタデータのパス |

`data/` と `*.duckdb` は `.gitignore` 済み（ユーザー生成物）。

## プロジェクト構成

```
vdas/                     解析エンジン（UI非依存）
  config.py               役割定義・パレット・パス
  db.py                   DuckDB 接続・スキーマ
  datasets.py             登録・スキーマ・役割の自動推定/永続化
  tags.py                 タグ CRUD・メンバーシップ
  analysis/
    core.py               PreparedData（時間軸正規化・dt・距離）
    metrics.py            示量/示強メトリクス、プール再計算
    gears.py              変速検出・滞在時間・遷移行列
    flags.py              0/1 間隔分析
    groups.py             コホート集計・比較・相対指数
  viz/charts.py           Plotly チャート（検証済みパレット）
app/
  Home.py                 エントリ
  pages/                  Datasets / Explore / Tags / Compare
scripts/gen_sample.py     J1939 風サンプル生成
sample_data/              生成済みサンプル parquet
```

## ライブラリ選定

- **Streamlit** … GUI にフレンドリーで、データアプリを素早く構築できる。
- **Plotly** … インタラクティブ（ズーム/パン/ホバー）で研究・業務用途に好適。
- **DuckDB** … parquet/csv を高速に直接クエリでき、タグ等の永続化も 1 ファイルで完結。
